# دليل لوحة تحكم موقع جاد العبدالله للمحاماة

هذا الدليل موجّه للموظف الذي سيتولى تحديث محتوى الموقع، بدون الحاجة لفهم الأكواد أو Git.

---

## 1. ما الذي تم إنشاؤه

تمت إضافة **لوحة تحكم محلية** (تعمل فقط على جهازك، لا يمكن لأي شخص خارج جهازك الوصول إليها) تتيح:

- تعديل بيانات التواصل (الهاتف، واتساب، البريد، العنوان، ساعات العمل).
- تعديل نصوص الصفحة الرئيسية وصفحة "من نحن" وبيانات قيادة المكتب.
- إضافة / تعديل / حذف الخدمات (مجالات الاختصاص).
- إضافة / تعديل / حذف القضايا، مع إمكانية رفع صورة لكل قضية.
- إضافة / تعديل / حذف مقالات المدونة.
- **زر واحد "حفظ ونشر التعديلات"** يقوم برفع كل التغييرات إلى GitHub تلقائيًا (بدون أي أوامر يدوية).

تصميم الموقع نفسه **لم يتغيّر** — فقط أصبح يقرأ بياناته من ملف بيانات بدلًا من أن تكون مكتوبة داخل الكود مباشرة.

---

## 2. التشغيل خطوة بخطوة

### المرة الأولى فقط:

1. تأكد من تثبيت [Node.js](https://nodejs.org) على الجهاز (نسخة 18 أو أحدث).
2. افتح مجلد المشروع، ثم افتح "موجه الأوامر" (PowerShell) داخله.
3. نفّذ الأمر التالي مرة واحدة فقط لتثبيت البرامج المطلوبة:

   ```powershell
   npm install
   ```

### في كل مرة تريد فتح لوحة التحكم:

1. نفّذ:

   ```powershell
   npm run admin
   ```

2. افتح المتصفح على:

   ```
   http://localhost:3000/admin
   ```

3. في أول مرة فقط، ستظهر لك صفحة **"إنشاء حساب الدخول"** — اختر اسم مستخدم وكلمة مرور (8 خانات على الأقل) واحفظهما في مكان آمن. لن تظهر هذه الصفحة مرة أخرى.
4. في المرات القادمة، ستظهر صفحة **تسجيل الدخول** مباشرة.

> لإيقاف لوحة التحكم: ارجع لنافذة الأوامر واضغط `Ctrl + C`.

> لمشاهدة الموقع نفسه دون فتح لوحة التحكم: نفّذ `npm run site` وافتح `http://127.0.0.1:4173` (الطريقة القديمة، ما زالت تعمل).

---

## 3. طريقة تغيير بيانات الدخول

- **نسيت كلمة المرور:** احذف الملف `data/admin-user.json` وأعد تشغيل `npm run admin`، ستظهر صفحة "إنشاء حساب الدخول" من جديد لإنشاء بيانات دخول جديدة.
- **تغيير كلمة المرور وأنت مسجل دخول:** من القائمة الجانبية اختر "إعدادات الحساب"، ثم أدخل كلمة المرور الحالية والجديدة.

---

## 4. طريقة استخدام لوحة التحكم

القائمة الجانبية تحتوي على الأقسام التالية:

| القسم | ماذا يفعل |
|---|---|
| بيانات التواصل | الهاتف، واتساب، البريد، العنوان، ساعات العمل — تظهر تلقائيًا في كل الموقع |
| النصوص التعريفية | نصوص الصفحة الرئيسية وصفحة "من نحن" |
| الخدمات / مجالات الاختصاص | إضافة/تعديل/حذف بطاقات الخدمات |
| القضايا | إضافة/تعديل/حذف القضايا مع رفع صورة لكل قضية |
| مقالات المدونة | إضافة/تعديل/حذف مقالات قصيرة |
| حفظ ونشر التعديلات | الزر الذي يرفع كل شيء على الإنترنت |
| إعدادات الحساب | تغيير كلمة المرور |

- كل حقل نصي يظهر مرتين: **عربي** و **English** — الموقع يدعم اللغتين، فيجب تعبئة الحقلين.
- عند حذف أي عنصر، ستظهر رسالة تأكيد قبل الحذف النهائي.
- عند رفع صورة لقضية: اختر الصورة، وستظهر معاينة لها فورًا، ثم اضغط "حفظ التعديلات" لتثبيتها.
- كل تعديل يُحفظ في ملف البيانات مباشرة، ويأخذ الموقع نسخة احتياطية تلقائية قبل كل حفظ (داخل مجلد `data/backups`) بحيث يمكن الرجوع لأي نسخة سابقة عند الحاجة.

---

## 5. طريقة النشر على الإنترنت (الأهم)

بعد الانتهاء من كل التعديلات المطلوبة:

1. اذهب لقسم **"حفظ ونشر التعديلات"** من القائمة الجانبية.
2. اضغط الزر **"حفظ ونشر التعديلات"**.
3. انتظر بضع ثوانٍ، وستظهر رسالة توضح ما حدث:
   - ✅ "تم الحفظ والنشر بنجاح" — التعديلات أصبحت منشورة.
   - ℹ️ "لا توجد تغييرات جديدة لنشرها" — لم تقم بأي تعديل جديد.
   - ❌ رسالة خطأ حقيقية إن حدثت مشكلة (مثل عدم وجود اتصال بالإنترنت).

لا حاجة لمعرفة Git أو GitHub — الزر يقوم بكل شيء تلقائيًا وبأمان.

---

## 6. الملفات التي تم إنشاؤها / تعديلها

**ملفات جديدة:**
- `data/site-data.json` — ملف بيانات الموقع (يقرأه الموقع تلقائيًا).
- `server/` — كود لوحة التحكم بالكامل (خادم + واجهة الإدارة).
- `package.json`, `package-lock.json` — لتثبيت البرامج المطلوبة عبر `npm install`.
- `.env.example` — نموذج إعدادات اختياري.
- `ADMIN_GUIDE.md` — هذا الدليل.

**ملفات تم تعديلها (بأقل قدر ممكن، دون كسر التصميم):**
- `script.js` — أصبح يقرأ المحتوى من `data/site-data.json` عند فتح الموقع، مع الاحتفاظ بنفس المحتوى الحالي كقيم افتراضية عند تعذر القراءة.
- `index.html` — تمت إضافة معرّفات (id) داخلية بسيطة لروابط الهاتف/البريد/واتساب في الفوتر لتحديثها تلقائيًا، دون أي تغيير في الشكل.
- `styles.css` — تمت إضافة تنسيق بسيط لعرض صورة القضية عند رفعها (لا يؤثر على أي تصميم موجود).
- `.gitignore` — تمت إضافة استثناءات لملفات الإعداد الحساسة (كلمة المرور المشفّرة، الجلسة، النسخ الاحتياطية) حتى لا تُرفع على GitHub.
- `README.md` — إضافة فقرة قصيرة تشير لهذا الدليل.

---

## 7. ملاحظات أمنية مهمة

- لوحة التحكم **لا تعمل إلا من نفس الجهاز** (localhost) — حتى لو كان الجهاز متصلاً بشبكة، لا يمكن لأي جهاز آخر فتحها.
- كلمة المرور تُحفظ **مشفّرة (Hash)** ولا تُحفظ كنص واضح في أي ملف.
- الملفات الحساسة (`data/admin-user.json`, `.env`, `data/.session-secret`, `data/backups/`) **لا تُرفع أبدًا** على GitHub (مستثناة عبر `.gitignore`).
- الصور المسموح رفعها فقط: `jpg`, `jpeg`, `png`, `webp`، وبحجم أقصى 5 ميجابايت، ويُعاد تسمية كل ملف تلقائيًا لمنع أي محاولة اختراق عبر اسم الملف.
- زر النشر لا يرفع أي شيء إذا لم تكن هناك تغييرات فعلية، ولا ينفّذ أي أوامر من إدخال المستخدم مباشرة (حماية من الأوامر الضارة).

---

## 8. حدود معروفة (بصراحة تامة)

- بعض التفاصيل الدقيقة جدًا في تصميم الصفحة الرئيسية (مثل الخلفيات الخاصة بكل خدمة في قسم "حلولنا") ليست قابلة للتعديل من اللوحة حاليًا لارتباطها المباشر بتصميم الصفحة — يمكن إضافتها لاحقًا إذا احتجتها.
- الخدمات الخمس الأساسية (تجاري، مالي، حقوقي، تنفيذي، عقاري) يُفضّل تعديل نصوصها فقط دون حذفها، لأنها مرتبطة ببعض روابط الموقع الثابتة (مثل روابط الفوتر). يمكن حذفها إذا رغبت، لكن الموقع سيستمر بالعمل بدون كسر حتى لو حذفتها بفضل قيم احتياطية مدمجة.
